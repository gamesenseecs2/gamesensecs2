## 🛠️ Troubleshooting Common Issues

**❌ Loader closes immediately after launch**
The loader requires administrator privileges and .NET 6.0 runtime. If it shuts down without an error window, your system may be missing the runtime.
**Solution:** Install .NET 6.0 Desktop Runtime from the official Microsoft page, then right-click `loader.exe` and choose **Run as administrator**. Gamesense CS2 should initialize normally.

**❌ Menu overlay does not appear in-game**
This usually happens when CS2 launches in fullscreen exclusive mode, which blocks external overlays.
**Solution:** Switch CS2 to **Fullscreen Windowed** in video settings. The Gamesense CS2 Menu requires windowed or borderless mode to render its overlay layer.

**❌ Antivirus deletes the loader file**
Because Gamesense CS2 Free uses kernel-level hooks, heuristic scanners may flag the binary as a threat.
**Solution:** Add the entire install folder (e.g., `C:\Tools\GamesenseCS2\`) to your antivirus exclusion list before extracting the archive.

**❌ Cloud sync shows "Authentication Failed"**
The cloud system requires a stable internet connection and a valid session token that refreshes every 24 hours.
**Solution:** Restart the loader to generate a fresh token. If the issue persists, delete `C:\Tools\GamesenseCS2\cache\token.dat` and relaunch Gamesense CS2 Menu.

**❌ FPS drops significantly with ESP enabled**
On older GPUs, rendering multiple ESP elements simultaneously can exceed the draw-call budget.
**Solution:** Disable skeleton ESP and reduce the maximum render distance in the Gamesense CS2 configuration panel. Keeping only box and health-bar ESP active usually resolves the issue.
